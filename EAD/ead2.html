Qual o objetivo da estrutura condicional IF - ELSE ? De um exemplo.

O objetivo da estrutura condicional if-else em JavaScript (ou qualquer outra linguagem de programação) é permitir que o programa tome decisões baseadas em uma condição específica. Ela permite que você execute diferentes blocos de código dependendo se a condição é verdadeira ou falsa.

1 let idade = 18 ;

2

3 if ( idade >= 18 ) {

4 ㅤ console.log ( " Você é maior de idade. " ) ;

5 } else {

6 ㅤ console.log ( " Você é menor de idade. " ) ;

7 }

Neste exemplo, o programa verifica a idade e imprime uma mensagem diferente dependendo se a pessoa é maior ou menor de idade.

Você também pode encadear várias cláusulas if-else para criar condições mais complexas. Neste caso, é possível utilizar a cláusula else if para testar outras condições antes de chegar na cláusula else.

De um exemplo de quando podemos usar a estrutura condicional SWITCH - CASE;

A estrutura condicional switch-case em JavaScript é usada quando você deseja testar uma expressão contra múltiplos valores possíveis e executar um bloco de código diferente para cada um desses valores. Ela pode ser uma alternativa mais concisa e legível do que várias cláusulas if-else encadeadas, especialmente quando você tem muitas condições para testar.

1 let diaDaSemana = 3 ;

2

3 switch ( diaDaSemana ) {

4 ㅤ case 1 :

5 ㅤㅤ console.log ( " Domingo " ) ;

6 ㅤㅤ break ;

7 ㅤ case 2 :

8 ㅤㅤ console.log ( " Segunda-feira " ) ;

9 ㅤㅤ break ;

10 ㅤ case 3 :

11 ㅤㅤ console.log ( " Terça-feira " ) ;

12 ㅤㅤ break ;

13 ㅤ case 4 :

14 ㅤㅤ console.log ( " Quarta-feira " ) ;

15 ㅤㅤ break ;

16 ㅤ case 5 :

17 ㅤㅤ console.log ( " Quinta-feira " ) ;

18 ㅤㅤ break ;

19 ㅤ case 6 :

20 ㅤㅤ console.log ( " Sexta-feira " ) ;

21 ㅤㅤ break ;

22 ㅤ case 7 :

23 ㅤㅤ console.log ( " Sabado " ) ;

24 ㅤㅤ break ;

25 ㅤㅤ default :

26 ㅤㅤ console.log ( " Dia inválido " ) ;

27 }

Uma coisa importante a se lembrar é que, se você omitir o break no final de um caso, o código continuará a ser executado no próximo caso. Isso pode ser útil em algumas situações, mas geralmente é considerado uma má prática, pois pode levar a resultados inesperados. Para evitar isso, você pode usar a instrução fallthrough para indicar que deseja que o código continue a ser executado no próximo caso.
